Apple release field guide.

A durable plan for taking Time from a private SwiftUI build to TestFlight, the App Store, or a notarized Mac download.

Recommendation

Build free. Pay when the release is ready.

Develop the iPhone and Mac apps with Xcode and use them personally first. Enroll in the Apple Developer Program one or two weeks before the first TestFlight build. Then use the paid year to ship both apps and learn the full production process.

Decision in one line

Publish the first version for experience, not revenue.

The existing production backend makes Time a low-risk vehicle for learning native architecture, code signing, TestFlight, privacy declarations, App Review, and release operations.

Budget

What Apple costs.

$0

Development

Xcode, simulator testing, Mac builds, and limited personal-device testing are free.

$99

Annual membership

One Apple Developer Program membership covers iOS and macOS apps. Apple charges local currency where available.

$0

Per submission

There is no separate fee for each build, TestFlight upload, App Review submission, or free-app download.

If membership expires: apps disappear for new downloads and updates stop. Existing installations continue working. Renewing restores free apps without another review.

Distribution choices

Three valid paths.

Recommended

App Store + Mac App Store

Maximum trust and résumé value. Requires annual membership, platform metadata, compliance work and App Review.

Mac alternative

Developer ID download

Publish a signed and notarized DMG through a website, GitHub or Homebrew. Avoids App Review, but still requires paid membership.

Private use

Local builds

Free and immediate. Mac builds run locally; free iPhone provisioning expires after seven days and needs periodic re-signing.

T3 Code’s model: direct distribution through its website, GitHub Releases and Homebrew rather than the Mac App Store. Professional direct downloads are Developer ID-signed and notarized; unsigned builds trigger Gatekeeper warnings.

Release checklist

From enrollment to approval.

  1. Choose individual or organization enrollment. Individual apps show the publisher’s legal name; organizations need a verified legal entity and D‑U‑N‑S number.
  2. Enroll in the Apple Developer Program. In India, complete enrollment through the Apple Developer app.
  3. Reserve the app name, register bundle identifiers, and create the iOS and macOS records in App Store Connect.
  4. Configure signing, production environments, associated domains, Auth0 callback URLs and the Convex production deployment.
  5. Add Sign in with Apple alongside Google login and provide in-app account deletion.
  6. Publish a privacy policy covering account data, timers, Auth0 and Convex; complete App Privacy and age-rating declarations.
  7. Archive and upload builds from Xcode, answer export-compliance questions, and test through TestFlight.
  8. Prepare icons, screenshots, description, support URL, review notes and a working reviewer account.
  9. Select the production build, submit it to App Review, resolve feedback, and choose the release timing.

Architecture direction

One product, shared native core.

  • SwiftUI multiplatform project with iOS and macOS targets.
  • Shared domain models, environment configuration, authentication state and Convex repository layer.
  • Official Convex Swift client for reactive queries and mutations.
  • Platform-specific navigation, commands, lifecycle behavior and presentation.
  • Production and development schemes pointed at separate Convex deployments.
